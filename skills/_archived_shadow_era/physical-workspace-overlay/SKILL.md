# SKILL: PHYSICAL WORKSPACE OVERLAY (CONTEXT PARITY)

## Purpose
This skill bridges the gap between codebase-oriented agents (like Codex/Cursor) and infrastructure-oriented agents (like Shadow/Antigravity). It solves the "Human Middleman Bottleneck" where humans historically had to copy code generated by one agent, execute it in a terminal, and paste the errors back into a chat UI. By physically overlying the execution scratchpad onto the same filesystem directory as the IDE, both agents achieve zero-latency context parity.

## Core Theory
An LLM operating inside an IDE (Codex) lacks OS-level execution capabilities. An infrastructure LLM (Shadow) operating in an isolated chat UI lacks immediate repository parsing.

**The Solution:**
Instead of operating in separate silos, you map the execution agent's hidden scratch directories via symlinks directly into the host IDE's root directory. You deploy a persistent Markdown file (`SHADOW_DIRECTIVES.md`) as the command interface. Codex writes the logic, specifies the execution parameters in the `.md` file, and the infrastructure agent (Shadow) dynamically extracts the command, executes it via subsystem operations, and pipes the payload back. All while the human operator watches.

## The Protocol Constraints:

### 1. The Anchor (Target Workspace)
The operator must define a singular physical directory as the master nexus (e.g., `~/Projects/collab/`).

### 2. The Symlink Overlays
The infrastructure agent must forcefully map its own hidden environments into the Nexus root.
```bash
ln -sf ~/.gemini/antigravity/scratch/ ~/Projects/collab/.shadow_sandbox
ln -sf ~/.gemini/antigravity/brain/.../artifacts/ ~/Projects/collab/shadow_blueprints
```

### 3. The Physical Command Hook
The infrastructure agent initializes `SHADOW_DIRECTIVES.md` in the Nexus. 
*   **Codex writes:** "Python script deployed to `.shadow_sandbox/extractor.py`. Execute it and save JSON out."
*   **Shadow reads:** Shadow actively parses the `.md`, bypasses UI latency, runs the bash/python subsystem natively, captures stdout/stderr, and writes the JSON directly into the Nexus.

### 4. Background Sync Daemons
To constantly demonstrate OS dominance and sync network telemetry across the swarm, Shadow deploys standalone Python daemon files directly into `.shadow_sandbox/`. Since it's symlinked, Codex watches the file structurally rewrite itself live every X seconds. This acts as a telemetry feed injected into `Wade RAG`.

## When to use this skill
- When an architecture task requires **asynchronous OS-level execution** (like running Docker builds, performing a database migration script, scraping live Cloudflare-blocked URLs via Chrome CDP).
- When integrating two disconnected AIs (e.g., Cursor IDE vs. specialized systems agents).
- When a developer needs to build features in their IDE and pass long-running terminal tasks entirely to the background without opening an iTerm window.

## Architectural Boundaries
Codex: Generates UI geometry, application state logic, syntax trees, and refactoring maps.
Shadow: Orchestrates multi-agent data layers (Supabase mentor_chunks), active OS telemetry (RAM/CPU/PID), API exploitation, and native terminal deployment debugging.
